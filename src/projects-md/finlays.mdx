# Overview
Finlays.xyz is a fully custom portfolio website built for a freelance client who works across Roblox development, lighting design, and graphic design. The core requirement was that the site had to be entirely manageable without touching any code — the client needed to be able to add new projects, update testimonials, and manage his commission status all from within the website itself.

To make that possible, I built a complete admin dashboard backed by Firebase, with a rich-text editor so the client could create and edit project entries as easily as writing a document.

## Why This Project?
My client needed a professional online presence to showcase his work and attract new commissions. As an independent freelancer working across multiple creative disciplines, a static or template-based site wouldn't cut it — he needed a living platform he could keep up to date himself, without relying on me for every small content change.

This was my second client-commissioned project, and it pushed me to think seriously about developer experience versus end-user experience: building something that is technically robust on the inside while being completely approachable for someone with no coding background on the outside.

## Features
- Dynamic projects and testimonials pages, with content fetched from Firestore.
- Full admin dashboard with sections for commission status, socials, projects, testimonials, and contact entries.
- Rich-text editor (TipTap) for creating and editing projects and testimonials without code.
- Contact form integrated with Resend for email delivery and Firestore for persistent storage.
- Firebase Authentication to protect the dashboard and restrict access to the client only.
- Parallax layout on the testimonials page for a polished visual feel.

## Tech Stack
- **Frontend:** Next.js, React.js, Tailwind CSS
- **Backend:** Firebase (Firestore, Authentication), Resend
- **Editor:** TipTap
- **Deployment:** Vercel, GitHub

## Challenges
### Firebase Authentication with Next.js
This was my first time using Firebase alongside Next.js, and the biggest hurdle was making authentication work correctly with server-side rendering. The issue came down to handling cookies and authentication tokens across page navigations. I solved this by implementing a custom cookie-based session system that stores the Firebase auth token and checks it on each request, keeping the client securely logged in without breaking the Next.js rendering model.

### Building a No-Code Editing Experience
The hardest design constraint was making the site fully editable by a non-technical user. After evaluating several options under school time pressure, I chose TipTap as the rich-text editor because it is headless and highly customisable. This let me tailor the editing experience to exactly what the client needed, and working with it gave me a solid understanding of how to integrate headless editor libraries into future React projects.
