# Overview
My client's instructions were to build a portfolio website he can use to display all his projects and client work on as an independent freelancer. He worked mainly with Roblox, lighting design, and graphic design. The main purpose was to be editable without looking through the source code of the project. This meant that I would need to utalise on-website building tools like a text editor and databases in order to make it possible to add projects effortlessly.

## Features
- Dynamic projects and testimonial pages, with data fetched from a database.
- Firebase integration (Firestore and Authentication)
- Contact Form which integrates with Resend and firestore to store contact entries.
- Fully working dashboard featuring sections for commision status, socials, projects, testimonials, and contact entries.
- Full rich-text editor using TipTap for projects and testimonials
- Parallax layout for testimonials page

## Tech Stack
- Frontend: Next.js, React.js, Tailwind CSS
- Backend: Firebase, Resend, TipTap
- Other Tools: GitHub, and Vercel

## Challenges
### Firebase Integration with Next.Js
This was my first time utilising Firebase alongside Next.Js. The main issue I ran into was working with authentications and cookies alongside each other. To overcome this issue, I created my own cookie solutions to store authentication tokens to ensure a user stays logged in after going to different pages on the website.
### Rich Text Editor for Projects and Testimonials
One big requirement of the project was to have it editable without code. This meant I had to integrate my own editor into the project. Due to time constraints with school, I did not have a massive amount of time to decide between which solution to go with for the project. In the end, I integrated TipTap into the project as it is headerless, very easy to customise and expand, and taught me how to integrate it potentially to other projects due to the flexibility of the tool.
